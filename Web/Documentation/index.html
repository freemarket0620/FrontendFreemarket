PiLab PLC Web Engineering App

This document describes the current Vue single-page application that replaced the original standalone HTML pages. It is written for developers who know Vue, JavaScript, and Tailwind-style UI work and need to extend, debug, or deploy the PiLab PLC web interface.

Architecture

The app is intentionally simple: Vue Router controls pages, the app shell controls the global frame, API modules wrap HTTP endpoints, and plcStore.js owns runtime polling.

Routing

The current router uses createWebHistory() and these routes:

export const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: '/', component: CommandCenter },
    { path: '/script', component: ScriptConsole },
    { path: '/editor', redirect: '/script' },
    { path: '/hmi', component: HmiDesigner },
    { path: '/tags', component: TagRegistry },
    { path: '/files', component: FileManager },
  ],
});
For final embedded deployment, hash routing may be easier unless the P4 web server redirects unknown paths back to /index.html. Use createWebHashHistory() if you want routes like /#/hmi.

PLC Store

src/stores/plcStore.js is the runtime data authority. Pages should read PLC runtime data from this store instead of creating independent pollers.

Command Center Polling

Polls /api/command_center at the configured cadence. This supplies overview data, script state, PLC mode, timings, masks, ticks, and heap information.

PLC Data Polling

Polls /api/plc_data only while a page/component has requested live tag data using usePlcData().

const store = usePlcStore();

// Runtime state
store.online
store.plcMode
store.plcRunning
store.flashWritesAllowed
store.overview
store.script
store.plcData
store.pointsByName
store.tagNames

// Actions
store.start();
store.usePlcData();
store.refreshCommandCenter();
store.refreshPlcData();
store.setPlcRun(true);
store.plcWrite('Q0', true);

Why usePlcData() Exists

Live tag polling is more expensive than high-level status polling. usePlcData() increments a subscriber count. When the last subscriber leaves, polling stops.

import { onMounted, onBeforeUnmount } from 'vue';
import { usePlcStore } from '../stores/plcStore';

const store = usePlcStore();
let releasePlcData = null;

onMounted(() => {
  releasePlcData = store.usePlcData();
});

onBeforeUnmount(() => {
  releasePlcData?.();
});
Best practice: any page that needs live point values should use usePlcData(). Pages that only need RUN/STOP/status should use the existing command-center store values.

Rule for New Endpoints

Add new firmware endpoints to an api/*.js module first, then call that helper from pages or stores. Avoid direct fetch() inside page components unless it is a one-off browser-native operation such as downloading a file.

Script Editor

The script editor is now a real Vue page using prism-code-editor-lightweight. It replaced the legacy standalone Prism textarea/layer editor.

Editor Features

  • Syntax-highlighted editing
  • Line numbers and keyboard editing behavior
  • Autocomplete popup
  • Snippets for common control blocks
  • Upload/status console
  • AngelScript diagnostics parser

Completion Sources

  • AngelScript keywords and types
  • Common snippets
  • Physical I/O names: I*, Q*, AI*, AO*
  • Live tags from plcStore.tagNames

Completion Module

src/editor/angelscriptCompletions.js keeps language and PLC suggestions separate from the editor component. Extend this file when adding new language features or PiLab built-ins.

// Example concept: add PiLab API symbols to the completion list
export function createAngelScriptCompletions({ liveTags = [] } = {}) {
  return [
    ...ANGELSCRIPT_KEYWORDS,
    ...PLC_IO_COMPLETIONS,
    ...liveTags.map(name => ({ label: name, type: 'variable' })),
  ];
}

HMI Designer

The HMI page is both a screen builder and a live operator screen. It uses local state for widget layout and shared PLC state for live values.

FeatureBehavior
Build/Live ModePersisted in local storage so route changes do not reset the mode.
Widget LayoutAuto-saved to local storage; defaults are only seeded when no saved layout exists.
Tag PickerUses live tags from /api/plc_data through the PLC store.
Toggle WritesWrites through store.plcWrite() / /api/plc_write.
Canvas LayoutFull-width engineering layout, not centered marketing layout.
Important: route changes unmount page components. Anything that must survive navigation should be kept in local storage, the PLC store, or a future dedicated HMI store.

Page Responsibilities

Command Center

Dashboard for controller state, runtime metrics, I/O masks, trend graphics, and RUN/STOP actions. It should not poll directly; it reads from plcStore.

Script Console

Editor, upload, diagnostics, and script status. It owns editor UI state but should reuse API wrappers and shared tag completions.

HMI Designer

Visual widget canvas. It consumes live tag data and persists design state locally until a firmware-backed save format is added.

Tag Registry

Loads/saves user-defined tags. These tags become available in PLC data and editor autocomplete.

File Manager

Manages LittleFS files. It uses shared PLC RUN/STOP/write-lock state to prevent writes during RUN mode.

How to Add a Page

  1. Create a page component in src/pages/NewTool.vue.
  2. Add a route in src/router.js.
  3. Add a navigation link in AppShell.vue.
  4. Use shared styles from style.css instead of creating a new visual system.
// src/router.js
import NewTool from './pages/NewTool.vue';

routes: [
  // ...existing routes
  { path: '/new-tool', component: NewTool },
]
<!-- AppShell.vue -->
<RouterLink to="/new-tool">New Tool</RouterLink>

How to Add API Calls

Keep firmware calls in src/api. Put shared runtime calls in plcStore only if multiple pages need them or if polling/caching/deduping is required.

// src/api/alarmApi.js
import { apiJson, apiPostJson } from './http';

export function loadAlarms() {
  return apiJson('/api/alarms');
}

export function acknowledgeAlarm(id) {
  return apiPostJson('/api/alarms/ack', { id });
}
If the endpoint is a live runtime stream or repeated poll, centralize it in a store. If it is a page-specific user action, a simple API helper is enough.

Recommended Symbol Manifest Wrapper

// C++ concept: register with AngelScript and metadata at the same time
RegisterFunction(
  "uint millis()",
  asFUNCTION(Millis),
  asCALL_CDECL,
  "Returns PLC uptime in milliseconds"
);

The web editor can then fetch /api/script_symbols and merge firmware API suggestions with language, I/O, and tags.

How to Add an HMI Widget

HMI widgets currently live inside HmiDesigner.vue. For now, add widget behavior there. Later, split each widget into a separate component.

  1. Add the widget type to controlTypes.
  2. Add default properties in defaults.
  3. Add render logic in the widget template.
  4. Add any inspector controls needed for the widget.
  5. Make sure values are persisted through the existing layout save format.
// Example concept
const controlTypes = ['gauge', 'readout', 'tank', 'led', 'trend', 'thermometer', 'toggle', 'valve'];

defaults.valve = baseProps({
  label: 'Valve',
  color: '#38bdf8',
  pin: 'Q0',
  active: false,
});
Long term, create src/hmi/widgets and register widget definitions instead of keeping all widget logic in one page.

Style Guide

The current visual direction is an engineering workstation, not a centered marketing page.

DoAvoid
Use full-width page layouts with shared left/right margins.Do not center large pages with boutique max-width layouts.
Use one global header and per-page local toolbars.Do not add extra page headers duplicating the global nav/status.
Use compact text links for navigation and secondary actions.Do not make every nav item a large button.
Use color for state and important actions.Do not color every element equally; it creates noise.
Keep industrial dark theme consistent.Do not introduce unrelated fonts, colors, or shadows per page.
Good rule: the header should almost disappear. The editor, HMI canvas, and runtime data should be the visual focus.

Local Development

Normal development uses Vite. The dev server can proxy/connect to the P4 depending on your environment configuration.

npm install
npm run dev

Build a production bundle:

npm run build

Preview the production build locally:

npm run preview
If npm hangs, check npm config get registry. It should use https://registry.npmjs.org/, not an internal or stale registry URL.

ESP32-P4 Deployment Notes

After npm run build, Vite creates a dist folder. Upload the contents of dist, not the folder itself, to the P4 web root.

dist/
  index.html
  assets/
    index-xxxx.js
    index-xxxx.css

Recommended Vite Setting

// vite.config.js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
  plugins: [vue()],
  base: './',
});

  • HMI widgets persist after navigation

Developer Principles

  1. One runtime truth: PLC state should come from the store, not random page pollers.
  2. Pages stay isolated: use routed pages, not one giant conditional component.
  3. API calls are wrapped: endpoints belong in src/api.
  4. Embedded constraints matter: avoid unnecessary polling, large dependencies, and excessive bundle size.
  5. Industrial UI over decorative UI: clarity, alignment, and state visibility beat visual novelty.
